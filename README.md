# novel-skill

一个可复用的中文长篇小说创作 Skill，用于规划、续写、逐章生成和自动校验。

这个仓库把小说创作流程拆成可执行的阶段化工作流，适合放进本地 AI skill 环境中反复使用，而不是只写一次性的短篇故事。

## 快速开始

把这个仓库作为本地 Skill 使用后，可以直接这样触发：

```text
使用 novel-skill 帮我写一部长篇中文小说
```

如果你已经有明确想法，也可以直接给出题材、主角和冲突，让 Skill 跳过部分前置问答：

```text
使用 novel-skill 写一部都市悬疑小说，主角是女法医，核心冲突是追查连环命案背后的旧案真相。
```

## 能做什么

`novel-skill` 主要解决的是“把一部长篇小说稳定写完”：

- 分阶段收集题材、主角、冲突、世界观和风格要求
- 自动生成人物档案、大纲和写作计划
- 按章节逐步写作，而不是一次性糊整本
- 支持中断后继续写
- 用脚本检查章节字数，发现不达标时回修

## 适用场景

这个 Skill 更适合这些任务：

- 想从零开始规划并写完一部长篇或中篇小说
- 已经有题材设定，但缺完整大纲和章节规划
- 想把小说写作过程拆成稳定可续写的文件化流程
- 想让 AI 不只是生成片段，而是持续写完整作品

不太适合这些任务：

- 只写一段几百字的灵感片段
- 一次性生成整本、完全不做章节控制
- 只做文学评论、设定讨论或脑暴，不进入正式创作

## 工作流

整个 Skill 分为 5 个阶段：

1. 初始化
2. 故事设定问答
3. 规划与确认
4. 逐章写作
5. 自动校验与修复

详细流程和写作指南都在 `references/` 目录里。

## 产出约定

Skill 默认在当前工作区下创建小说项目目录：

```text
./chinese-novelist/
```

典型产出文件包括：

- `00-人物档案.md`
- `01-大纲.md`
- `02-写作计划.json`
- `第01章-*.md`
- `第02章-*.md`

示例输出结构：

```text
chinese-novelist/
└── 20260503-悬案回声/
    ├── 00-人物档案.md
    ├── 01-大纲.md
    ├── 02-写作计划.json
    ├── 第01章-雨夜尸检.md
    ├── 第02章-缺席的证人.md
    └── 第03章-旧档案里的名字.md
```

## 脚本

检查单章字数：

```bash
python3 scripts/check_chapter_wordcount.py <chapter-file>
```

检查整部小说所有章节：

```bash
python3 scripts/check_chapter_wordcount.py --all <project-dir>
```
